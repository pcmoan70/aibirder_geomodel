# Test Plan: Norway + Nordic Residual GeoModel

**File:** `test_plan_20260427.md`
**Date:** 2026-04-27
**Status:** QA, validation, and acceptance plan

---

## 1. Goal

Verify that the model:

1. trains reproducibly,
2. uses all Nordic data correctly,
3. uses Norway-only features only inside Norway,
4. improves Norway predictions,
5. does not create cross-border artifacts,
6. handles altitude and non-harmonic week dependence,
7. uses flora/insect/geology features without leakage,
8. exports and runs reliably.

## 3. Unit tests

### 3.1 Encoders

Test:

```text
CoordinateEncoder
AltitudeEncoder
HybridWeekEncoder
CountryEncoder
CommonEnvEncoder
NorwayFeatureEncoders
BioticIndicatorEncoder
GeologyMineralEncoder
```

Checks:

```text
correct output shape
no NaNs
supports batch size 1 and large batches
works with missing masks
deterministic in eval mode
gradients flow
```

### 3.2 Week encoder tests

Tests:

```text
week index range 0–47
cyclic wraparound works
week 0 and week 47 are near in Fourier part
learned embedding can differ
smoothness loss handles wraparound
migration windows reduce smoothness penalty
```

### 3.3 Altitude tests

Tests:

```text
normalization correct
clipping correct
missing altitude handled
source flag handled
altitude dropout only active in training
```

### 3.4 Norway residual tests

Tests:

```text
Norway branch active for NO only
Norway branch inactive outside NO
missing Norway features make gate small or branch neutral
residual shape equals species logit shape
species-level gate has valid range [0, 1]
```

### 3.5 Smoothing tests

Tests:

```text
neighbor weights positive
weights decrease with distance
weights decrease with altitude difference
weights decrease with habitat/geochemistry/biotic difference
loss is zero when logits identical
loss is finite for random logits
```



### 6.3 Acceptance thresholds

The final model should satisfy:

```text
Norway validation improves over shared Nordic baseline
held-out region performance improves or is neutral
calibration is not materially worse
roughness decreases or remains controlled
border discontinuity does not increase materially
non-Norway performance does not degrade materially
```

Do not accept a model that only improves random validation but worsens spatial block validation.


## 15. Failure responses

If Norway residual overfits:

```text
increase residual L2
reduce residual rank
increase residual smoothing
reduce gate prior
disable local spatial residual
improve effort controls
```

If model over-smooths:

```text
decrease smoothing lambdas
use stronger ecological similarity weighting
reduce smoothing across sharp habitat/geology boundaries
```

If biotic indicators leak effort:

```text
cross-fit features
increase effort correction
drop high-effort-biased groups
replace raw indicator scores with habitat model predictions
```

If altitude response is unstable:

```text
increase altitude dropout
smooth altitude response
check DEM/GPS mismatch
use altitude source flag
```

If week curves oscillate:

```text
increase week smoothness
increase curvature penalty
reduce learned week dimension
```

## References and source notes

The design is intentionally independent of one exact upstream code commit, but it assumes the current public BirdNET GeoModel shape:
a multi-task neural network using raw latitude/longitude/week inputs, internal circular encoding, multi-label species classification,
an auxiliary environmental regression head, and an optional habitat head that maps predicted environmental features to species logits.
